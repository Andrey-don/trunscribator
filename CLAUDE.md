# Транскрибатор

Десктопное Python-приложение для транскрибации видео (MP4 и других форматов) и автоматического создания скриншотов по ключевым кадрам (scene detection).

## Стек

- **GUI**: `customtkinter` — современный tkinter-виджеты
- **Транскрибация**: `openai-whisper` — локальная модель, без API-ключей
- **Scene detection**: `scenedetect` (PySceneDetect) с `ContentDetector`
- **Видео**: `opencv-python` — извлечение кадров

## Структура проекта

```
trunscribator/
  main.py          # Точка входа, весь код приложения
  requirements.txt # Python-зависимости
  CLAUDE.md        # Документация проекта
```

## Запуск

```bash
pip install -r requirements.txt
python main.py
```

### Системные зависимости

**ffmpeg** обязателен для работы Whisper:
```bash
winget install Gyan.FFmpeg
```
После установки перезапустить терминал, чтобы `ffmpeg` появился в PATH.

## Как работает

1. Пользователь выбирает видеофайл через диалог.
2. Выбирает модель Whisper (`tiny` → `large`, баланс скорость/качество).
3. Нажимает «Обработать» — приложение в фоновом потоке:
   - Транскрибирует аудио через Whisper → сохраняет `.txt`
   - Определяет смены сцен через `ContentDetector` (порог 27.0) → сохраняет скриншоты `1.jpg`, `2.jpg`, ...
4. Если сцены не найдены — запасной вариант: скриншот каждые 30 секунд.

## Вывод

Результаты сохраняются рядом с видеофайлом в папку с именем видео:

```
папка_с_видео/
  название_видео/
    название_видео.txt   ← транскрипция
    1.jpg                ← первый кадр первой сцены
    2.jpg
    ...
```

## Ключевые решения

- **Фоновый поток** (`threading.Thread`) — GUI не замерзает во время обработки.
- **`self.after(0, fn)`** — все обновления UI из фонового потока через очередь tkinter.
- **Порог ContentDetector = 27.0** — эмпирически подобранный баланс: не реагирует на мелкие изменения освещения, но ловит монтажные склейки.
