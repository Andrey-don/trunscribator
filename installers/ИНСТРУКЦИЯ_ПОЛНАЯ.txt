Приложение «Транскрибатор» — это десктопная программа для Windows,
которая автоматически переводит речь из видео в текст, делает
скриншоты по смене сцен и создаёт HTML-отчёт с содержанием.

Дополнительно встроена интеграция с Ollama — локальным ИИ,
который исправляет ошибки распознавания, делает краткое содержание
и выделяет ключевые моменты видео — без интернета и без API-ключей.

ТРЕБОВАНИЯ К КОМПЬЮТЕРУ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Операционная система:  Windows 10 или Windows 11 (64-бит)
  Оперативная память:    минимум 8 ГБ (рекомендуется 16 ГБ)
  Видеокарта:            NVIDIA желательна (для ускорения Ollama)
                         без видеокарты тоже работает, но медленнее
  Место на диске:        минимум 20 ГБ свободного места
  Интернет:              нужен только для Варианта 1
                         Вариант 2 работает полностью без интернета
                         (кроме скачивания самого приложения ~50 МБ)

ВАРИАНТ 1 — ОНЛАЙН-УСТАНОВКА (нужен интернет)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Используйте этот вариант если на целевом компьютере есть
быстрый интернет и достаточно времени (~30-60 минут).

Что скачивается автоматически:
  • Python 3.11             (~25 МБ)
  • ffmpeg                  (~80 МБ)
  • Приложение с GitHub     (~50 МБ)
  • Python-зависимости      (~3 ГБ, включая Whisper и PyTorch)
  • Ollama                  (~50 МБ)
  • Модель gemma3:12b       (~8 ГБ)
  Итого трафика:            ~11-12 ГБ

КАК УСТАНОВИТЬ:
  1. Скопируйте файл online\install_online.bat на целевой компьютер
     (можно на рабочий стол или в любую папку)
  2. Запустите install_online.bat двойным кликом
  3. Если Windows спросит "Разрешить изменения?" — нажмите "Да"
  4. Следуйте инструкциям на экране
  5. После завершения на рабочем столе появится ярлык "Транскрибатор"

  ВАЖНО: После установки Python скрипт попросит перезапустить его —
  это нормально. Закройте окно и запустите install_online.bat снова.

ВАРИАНТ 2 — УСТАНОВКА С НОСИТЕЛЯ (без скачивания моделей)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Используйте этот вариант для переноса на компьютер без интернета
или с медленным соединением. Модели (~13 ГБ) уже на этом носителе.

Что скачивается даже в этом варианте (~50 МБ, нужен интернет):
  • Само приложение с GitHub

Что берётся с носителя (интернет не нужен):
  • Модель Ollama gemma3:12b    (~8 ГБ) из папки ollama_models\
  • Модели Whisper              (~5 ГБ) из папки whisper_models\

КАК УСТАНОВИТЬ:
  1. Скопируйте с носителя на целевой компьютер ВСЮ папку hybrid\
     целиком (вместе с ollama_models\ и whisper_models\)
     Например, на рабочий стол или в папку Загрузки.

     Структура должна быть такой:
       hybrid\
         install_hybrid.bat
         ИНСТРУКЦИЯ.txt
         ollama_models\
           blobs\
           manifests\
         whisper_models\
           base.pt
           medium.pt
           small.pt
           large-v3.pt

  2. Откройте папку hybrid\ и запустите install_hybrid.bat
  3. Если Windows спросит "Разрешить изменения?" — нажмите "Да"
  4. Установщик проверит наличие папок с моделями и сообщит о результате
  5. Следуйте инструкциям на экране
  6. После завершения на рабочем столе появится ярлык "Транскрибатор"

  ВАЖНО: После установки Python скрипт попросит перезапустить его —
  закройте окно и запустите install_hybrid.bat снова.

ЧТО УСТАНАВЛИВАЕТСЯ И КУДА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Приложение:      C:\Users\<имя>\Trunscribator\
  Модели Whisper:  C:\Users\<имя>\.cache\whisper\
  Модели Ollama:   C:\Users\<имя>\.ollama\models\
  Ollama:          C:\Users\<имя>\AppData\Local\Programs\Ollama\
  Ярлык запуска:   Рабочий стол → "Транскрибатор"

КАК ПОЛЬЗОВАТЬСЯ ПРИЛОЖЕНИЕМ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. Запустите приложение через ярлык "Транскрибатор" на рабочем столе

  2. Нажмите "Выбрать видео" и укажите файл (MP4, MKV, AVI, MOV)

  3. Выберите модель Whisper:
       tiny   — самая быстрая, качество ниже
       base   — быстро, среднее качество
       small  — хорошее качество (рекомендуется для начала)
       medium — высокое качество, медленнее в ~4 раза
       large  — максимальное качество, самая медленная

     ПЕРВЫЙ ЗАПУСК: при выборе новой модели она автоматически
     скачивается (~145 МБ для base, ~1.5 ГБ для medium).
     Это происходит только один раз.
     Если вы используете Вариант 2 — модели уже скопированы
     и скачиваться не будут.

  4. Слайдер "Текст каждые" — как группировать текст:
       0 сек = каждый отдельный фрагмент речи
       10 сек = блоки по 10 секунд (по умолчанию)
       30 сек = крупные блоки

  5. Слайдер "Скриншотов" — количество скриншотов:
       мало = только крупные смены сцен
       много = скриншот при каждом изменении кадра

  6. Переключатели:
       Скриншоты   — делать ли скриншоты из видео
       HTML-отчёт  — создавать ли итоговый документ

  7. Переключатель "Улучшить текст (Ollama)":
       Включите чтобы ИИ исправил ошибки распознавания.
       После включения появятся дополнительные опции:
         Модель Ollama    — название модели (по умолчанию gemma3:12b)
         Краткое содержание — ИИ напишет резюме видео
         Ключевые моменты   — ИИ выделит важные моменты с таймкодами

  8. Нажмите "Обработать" и дождитесь завершения.
     Для 10-минутного видео на модели small: ~5-10 минут.
     С Ollama добавляется ещё ~30-60 секунд.

  9. По завершении автоматически откроется:
       • Папка с результатами в проводнике
       • HTML-отчёт в браузере

РЕЗУЛЬТАТЫ ОБРАБОТКИ

  Результаты сохраняются рядом с видеофайлом в папку:
    <папка_с_видео>\<название_видео>_v2\

  Содержимое папки:
    название_видео.txt   — транскрипция с таймкодами
    index.html           — HTML-отчёт (открыть в браузере)
    00_00_00.jpg         — скриншоты (формат: ЧЧ_ММ_СС)
    00_01_30.jpg
    ...

ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ПРОБЛЕМА: Приложение не запускается
  РЕШЕНИЕ:  Убедитесь что установлен Python. Запустите install снова.

  ПРОБЛЕМА: Ошибка "ffmpeg not found"
  РЕШЕНИЕ:  Установите ffmpeg через winget:
            winget install --id Gyan.FFmpeg
            Перезапустите компьютер.

  ПРОБЛЕМА: Транскрипция пустая или с ошибками
  РЕШЕНИЕ:  Попробуйте более крупную модель (small → medium → large)

  ПРОБЛЕМА: Ollama не работает / ошибка подключения
  РЕШЕНИЕ:  Проверьте что Ollama запущена — в трее (правый нижний
            угол экрана) должна быть иконка Ollama.
            Если нет — откройте Ollama из меню Пуск.

  ПРОБЛЕМА: Краткое содержание на английском или китайском
  РЕШЕНИЕ:  Это известная особенность модели. Попробуйте снова —
            обычно со второй попытки отвечает на русском.
